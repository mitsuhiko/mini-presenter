# Current Slide Preview

Add an iframe in the presenter view that shows the current slide, synced with the display.

## Approach

The presenter view loads the presentation in an iframe and navigates it to match the display's current hash.

```
┌─────────────────────────────────────┐
│  Presenter View                     │
│  ┌───────────────────────────────┐  │
│  │  <iframe src="/?hash#slide">  │  │
│  │  (current slide preview)      │  │
│  └───────────────────────────────┘  │
└─────────────────────────────────────┘
```

## Implementation

### Sync Logic
1. Presenter receives state update: `{ slideId: '#slide-3', hash: '#slide-3' }`
2. Update iframe src or navigate iframe to that hash
3. Handle iframe load events to avoid flicker

### Considerations
- The iframe will also receive the injected script
- Need to prevent the iframe from registering as a display
- Add query param: `/?_presenter_preview=1` → injected script skips registration
- Or: iframe sends state but marked as "preview" role

### CSS
- Iframe should maintain presentation aspect ratio
- Scale down to fit presenter view
- Possible: add zoom/scale transform
