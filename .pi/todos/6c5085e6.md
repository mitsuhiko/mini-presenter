# Remote URL Support (Follow-up)

Support presenting remote URLs (not just local files) by loading them in an iframe wrapper.

## CLI Interface

```bash
mini-presenter https://example.com/slides/
mini-presenter --remote https://example.com/slides/
```

## Architecture

Since we can't inject scripts into remote pages, use an iframe wrapper:

```
Server serves: wrapper.html
                   │
                   ▼
        ┌──────────────────────┐
        │  wrapper.html        │
        │  ┌────────────────┐  │
        │  │ <iframe>       │  │
        │  │ remote URL     │  │
        │  └────────────────┘  │
        │  + injected.js       │
        └──────────────────────┘
```

## Implementation

### client/wrapper.html
```html
<!DOCTYPE html>
<html>
<head>
  <style>
    iframe { width: 100%; height: 100%; border: none; }
  </style>
</head>
<body>
  <iframe id="presentation" src="REMOTE_URL"></iframe>
  <script src="/_/client/wrapper.js"></script>
</body>
</html>
```

### client/wrapper.js
- Connect to WebSocket as display
- Forward keyboard events to iframe via postMessage
- Try to detect hash changes (limited by cross-origin)

## Limitations

### Cross-Origin Restrictions
- Cannot read iframe's location.hash
- Cannot inject scripts
- Can only send postMessage (if page listens)
- Can dispatch keyboard events to iframe

### Workarounds
- If same origin: full access
- If cross-origin: keyboard-only control, no state detection
