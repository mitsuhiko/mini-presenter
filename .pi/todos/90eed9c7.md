# Configuration File Support

Support an optional `presenter.json` in the presentation directory for customization.

## File Location
```
presentation/
├── index.html
├── presenter.json    ← configuration
└── notes/
```

## Schema

```json
{
  "title": "My Presentation",
  
  "keyboard": {
    "next": ["ArrowRight", "Space", "PageDown", "l", "j"],
    "prev": ["ArrowLeft", "PageUp", "h", "k"],
    "first": ["Home"],
    "last": ["End"]
  },
  
  "notes": {
    "source": "files"  // "api" | "files" | "none"
  },
  
  "slides": [
    "#intro",
    "#slide-1",
    "#slide-2"
  ]  // Optional: explicit slide order for next preview + PDF export
}
```

## Usage

### Keyboard Customization
- Server sends config to presenter on connect
- Presenter uses config for keyboard shortcuts
- Display uses config for which keys to listen for (future)

### Slide Order
- If provided, enables next slide preview
- Enables accurate PDF export
- Without it, rely on hash observation

## Server Changes

1. On startup, check for `presenter.json`
2. Parse and validate
3. Send to presenter via WebSocket on connect
4. Use for keyboard mapping when forwarding commands

## API

```
GET /_/api/config
→ { "title": "...", "keyboard": {...}, ... }
→ { }  // if no config file
```
