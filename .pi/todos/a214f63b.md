# Next Slide Preview (Follow-up)

Add preview of the next slide in the presenter view.

## Challenge

Without knowing the slide order, we can't predict the next slide's hash.

## Approaches

### Option 1: Dual Iframes
- Presenter loads presentation twice
- One iframe at current hash, one at "next" hash
- Need slide order from config or API

### Option 2: Presentation API
```javascript
window.miniPresenter.getSlideList()  // → ['#1', '#2', '#3']
// or
window.miniPresenter.getNextSlide()  // → '#3'
```

### Option 3: Config File
```json
{
  "slides": ["#intro", "#slide-1", "#slide-2", "#conclusion"]
}
```

### Option 4: History Tracking
- Remember all hashes seen during navigation
- Build slide order dynamically
- Unreliable for first run

## Recommended Approach

1. Check config for `slides` array
2. If not present, check API for `getSlideList()`
3. If neither, disable next preview (show placeholder)
